# Shoot Protocol — 현장 촬영 운영 기준

VP Supervisor가 현장에서 즉각 판단을 내리는 기준. 모든 판단 기록은 `/take-log`로 관리.

## 테이크 판정 기준

| 판정 | 코드 | 정의 | 다음 액션 |
|---|---|---|---|
| **OK** | `OK` | 사용 가능한 테이크 — 연기, 카메라, 기술 모두 수용 가능 | 다음 테이크 또는 씬 완료 |
| **NG** | `NG` | 사용 불가 — 아래 NG 분류 참조 | 원인 파악 후 재촬영 |
| **HOLD** | `HOLD` | 보류 — 포스트 확인 필요, 촬영은 계속 | VFX/편집팀 검토 후 최종 판정 |

### NG 분류 (--dept 태그)

| 부서 | 코드 | 원인 예시 |
|---|---|---|
| 기술 | `tech` | FPS 드랍, 동기화 실패, 트래킹 오류 |
| 아트 | `art` | 에셋 팝핑, 텍스처 스트리밍 딜레이, 배경 불일치 |
| MoCap | `mocap` | 마커 오클루전, 리타겟팅 오류, 레이턴시 스파이크 |
| 연기 | `acting` | 감독 판단 NG (VP Supervisor 범위 외) |
| LED | `led` | 색감 불일치, 모아레, 패널 불량 |

## 블로커 정의

### P1 블로커 (촬영 즉시 중단)
촬영 중단 기준 — 하나라도 해당하면 즉시 `/risk-flag` 발동:
- Genlock/Timecode 체인 완전 실패 (FOptGenlockMonitor/FOptTimecodeMonitor 레드)
- FPS가 목표의 50% 이하로 드랍 (24fps 기준 12fps 이하)
- 전체 LED 패널 1/4 이상 불량
- MoCap 카메라 3대 이상 동시 실패 (캘리브레이션 무효화)
- UE 에디터 크래시

### P2 블로커 (해당 씬 촬영 보류)
5분 내 해결 불가 시 씬 교체 또는 휴식:
- 특정 배우 MoCap 트래킹 지속 실패
- 텍스처 스트리밍 지속 딜레이 (> 3연속 NG)
- LED 패널 단일 섹션 불량 (씬 구도에 영향)

### P3 블로커 (다음 씬 전 해결)
현재 씬은 계속, 다음 씬 전 조치:
- 에셋 팝핑 (소수 에셋, 영향 미미)
- 텍스처 해상도 육안 인식 가능한 차이
- Ambient Lockit 배터리 경고

---

## 5연속 NG 자동 경고

`/take-log`가 같은 씬에서 NG를 5회 연속 기록 시 자동으로:
1. 콘솔에 `⚠️ 5연속 NG — 블로커 판단 필요` 경고 출력
2. `takes.md`에 `## ⚠️ 블로커 경고` 섹션 자동 추가
3. 가장 많은 NG 원인 부서 자동 집계 → 해당 팀 액션 아이템 제안

## 리스크 대응 매트릭스

| 리스크 유형 | 심각도 | 즉각 조치 | 에스컬레이션 기준 |
|---|---|---|---|
| FPS 드랍 | P1 | Optimazation 플러그인 CVar 조정 → FAIL이면 해당 씬 NO-GO | 3분 내 복구 불가 |
| Genlock 실패 | P1 | SPG 8000 → eSync 체인 재확인, `/sync-check` 재실행 | 10분 내 복구 불가 |
| MoCap 오류 | P2 | Motive 재보정, 마커 확인 | 해당 배우 3테이크 연속 실패 |
| LED 불량 | P2 | 패널 교체 요청, 구도 조정 고려 | 씬 1/4 이상 영향 |
| 타임코드 드리프트 | P2 | Tentacle / Ambient Lockit 재동기화 | Drift > 2프레임 |
| 네트워크 지연 | P3 | 스위치 확인, 불필요 트래픽 차단 | LiveLink 레이턴시 > 50ms |

## 다운타임 기준

다운타임은 `/kpi-report`의 Efficiency KPI에 반영된다.

| 상태 | 다운타임 계산 | 분류 코드 |
|---|---|---|
| 기술 장비 점검 중 | 시작~복구 완료 | `DT_TECH` |
| 조명/LED 조정 | 시작~조정 완료 | `DT_LIGHT` |
| 배우 준비 중 | VP Supervisor 범위 외 | — |
| 브레이크 | VP Supervisor 범위 외 | — |

허용 다운타임 기준:
- 기술 원인 다운타임 < 15% (8시간 촬영 기준 72분 이내)
- 단일 장애 복구 시간 < 30분 (초과 시 대안 플랜 발동)

## 씬 완료 기준 (Scene Wrap)

다음 조건 충족 시 감독과 협의 후 Scene Wrap 선언:
- OK 테이크 ≥ 2개 (동일 컷 기준)
- HOLD 테이크가 있는 경우 포스트팀 동의 확인
- 해당 씬의 모든 컷 촬영 완료

Scene Wrap 후 `/data-wrangle`로 메타데이터 구조화.
